# VITAO Roadmap

## Phase 0 — Repository Foundation

Goal: prepare the repository so Codex can work with clear product, design, and implementation context.

Deliverables:

- README
- AGENTS.md
- Product brief
- Design direction
- Roadmap
- Codex task list

## Phase 1 — Static MVP Foundation

Goal: create a deployable static React app.

Deliverables:

- Vite + React + TypeScript app
- clean folder structure
- reusable layout components
- basic routing
- static product data
- initial styles/design tokens
- GitHub Pages build configuration

Pages:

- Home
- Products
- Product Detail
- Custom Order
- About
- Contact + FAQ

Success criteria:

- `npm run build` passes
- app can be deployed to GitHub Pages
- visual direction is recognizable
- no backend/payment/cart/auth added

## Phase 2 — Premium UI Implementation

Goal: implement the visual concept as polished responsive pages.

Deliverables:

- premium header and footer
- home page sections
- product catalog grid
- product detail layout
- custom order page
- about page
- contact + FAQ page
- responsive mobile layout

Success criteria:

- desktop layout looks premium and coherent
- mobile layout is clean and usable
- reusable components exist for repeated UI patterns

## Phase 3 — Content and Product Polish

Goal: make the site feel ready for first real visitors.

Deliverables:

- 8–12 realistic placeholder products
- refined product names and descriptions
- consistent product categories
- better CTAs
- FAQ content
- material and process content
- contact/order request flow

Success criteria:

- site can be shared with potential customers
- visitors understand what to do next
- custom order CTA is obvious

## Phase 4 — Launch Preparation

Goal: prepare for public MVP validation.

Deliverables:

- GitHub Pages deployment
- basic SEO metadata
- Open Graph image
- favicon / minimal brand assets
- analytics placeholder or lightweight analytics plan
- manual QA checklist

Success criteria:

- public URL works
- no broken navigation
- no obvious mobile issues
- custom order/contact path works

## Later / Not MVP

Do not implement yet unless explicitly requested:

- backend
- database
- admin panel
- full cart
- checkout
- payment provider
- user accounts
- CMS
- product reviews
- complex filters
- inventory management
